# Shared database antipattern

Этот подход является антипаттерном, потому что:

1. нарушает принцип [[Information hiding]]
1. вроде бы перешли на микросервисы, а все равно не получили независимость разработки - все микросервисы связаны через БД, значит нельзя понять какие изменений БД безопасные, а какие нет. Нет явных версионируемых контрактов, через которые можно реализовать интеграцию
1. непонятно кто владеет данными: бизнес логика какого микросервиса контролирует соблюдение инвариантов
